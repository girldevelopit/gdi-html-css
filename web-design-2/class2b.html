The Position Property

Position

The position property defines how and where an element can move on the page. It has five values and five coordinate properties:

  • Static
  • Fixed
  • Absolute
  • Relative
  • Sticky
  • Top
  • Bottom
  • Left
  • Right
  • Z-Index

position: static

This is the default position of all elements; the normal document flow

position: fixed

  • An element is fixed to a position relative to the browser window
  • The element is also moved out of the document flow
  • Coordinates can be applied for precise positioning

.box-three {
  position: fixed;
  top: 10px;
  left: 0;
}

position: absolute

  • The element is positioned relative to its parent container
  • The parent container must have a position value of relative, or the element will be positioned relative to the closest parent with a non-static value
  • The element is removed out of the normal document flow
  • Coordinates can be applied for precise positioning

.box-two {
  position: absolute;
  top: 0;
}

position: relative

  • An element is positioned relative to itself
  • It does not move out of the document flow
  • Coordinates can be applied for precise positioning

.box-five {
  position: relative;
  right: 100px;
}

position: sticky

  • The element is fixed and relative to a parent container until the parent container moves out of the viewport
  • Browser support is not at 100%; use with care

.box-one {
  position: sticky;
  top: 10px;
}

Z-Index

  • The z-index property defines the order of how elements overlap each other
  • The default value of an element's z-index is auto or 0
  • An element with a high z-index will stack over elements with lower z-index values
  • Only works on elements that have a non-static position

.box-four {
  position: sticky;
  top: 10px;
}

CSS Cascade, Specificity & Inheritance

CSS Cascade

The cascade is how CSS determines what CSS rule or declaration takes precedence when multiple stylesheets are in use; or when more than one rule applies to a given element.

Origin of CSS Declarations

The browser reads CSS from three sources:

  • User Agent stylesheet (Browser defaults)
  • Author stylesheet (Our site's custom styles)
  • User stylesheet (CSS set by the user)

CSS Cascade (MDN)

Stylesheet Order of Importance

CSS calculates the importance of declarations from the three types of stylesheets from most to least importance:

  • User !important declarations
  • Author !important declarations
  • Author declarations
  • User declarations
  • Default browser declarations

!important

The !important flag in CSS means a style
will override any other declaration 

p {
  color: yellow !important;
}

CSS Specificity

CSS is also calculating and comparing the weight or specificity of the rules in our stylesheets;
from most to least specific:

  • Inline CSS
  • IDs
  • Classes, Pseudo-classes, Attribute selectors
  • Elements, Pseudo-elements

Specificity Tips

  • 1 ID selector out weighs 100+ class selectors
  • 1 class selector outweighs 100+ element selectors
  • The Universal selector * has no specificity
  • Code architecture changes: Choose specificity over order
  • List your main stylesheet last
  • !important only as a last resort

CSS Inheritance

  • Some CSS property values set on parent elements
    are inherited by their child elements, and some are not
  • Understanding inheritance can help us write fewer, less redundant code

CSS Inheritance (MDN)

Inheritable Properties

Font properties, color, text alignment are common
properties can be inherited by default when a child
element has no declared values for these properties 

.parent {
  color: red;
  font-size: 20px;
  text-align: center;
}

.child {
}

CSS Inheritance

We can force inheritance with the inherit keyword

.parent {
  color: red;
  border: 1px solid black;
}

.child {
  border: inherit;
}

CSS Inheritance
