The Display "Inner" Properties

Grid, Flex

Their values affect the elements inside of the element that the style is applied to.

display: flex;

Use it on the parent element to position its immediate children.

display: flex;
align-items: center;
justify-content: space-between;

display: flex;

The following are a few of Flexbox's Greatest Hits.

display: flex;

Properties for the parent of flex

  • justify-content (horizontal alignment)
  • align-items (vertical alignment)
  • flex-wrap
  • gap
  • flex-direction

display: flex;

Properties for children of flex

  • order
  • flex-direction
  • align-self
  • justify-self
  • flex-basis
  • flex: none

display: flex;

Flexbox vs. Grid

Flexbox and all other layout methods are only 1 dimensional

This can lead to unexpected results

The Evolution of Grid Frameworks

To achieve a grid layout in the past required complex hacks using the only CSS that was available at the time, never meant to do more than layout basic documents. These frameworks were once standard and really useful.

Popular frameworks: Bootstrap, Foundation, Bulma, Materialize, Skeleton, Neat, Suzy, Simple Grid

The Evolution of Grid Frameworks

Grid Frameworks rely on specific markup structure and class names to achieve layout. At first they used floats or inline-block. Later they used Flexbox. Still, none of these CSS properties were meant to lay out content in this way.

CSS Grid was built specifically to solve that problem.

CSS Grid allows you to lay out elements in 2D, rows & columns

Placing items on the grid

display: grid;
