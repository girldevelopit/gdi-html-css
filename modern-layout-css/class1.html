The Display "Outer" Properties

Inline, Block, Inline-block, None

These values all affect the element that the style is applied to.

In the beginning...

In HTML, there were block & inline elements.

Block vs. Inline

Block: Layout vertically, one below the other

Inline: Display one after another in the direction that sentences run.

display: block;

A few elements with block as a default

  • <div>
  • <p>
  • <form>
  • <header>
  • <footer>
  • <h1><h2><h3><h4><h5><h6>
  • <main>
  • <section>
  • <ul><ol>
  • <li>

display: block;

Often used as containers

display: block;

Takes margin & padding on all sides

display: inline;

A few elements with inline as a default

  • <span>
  • <a>
  • <button>
  • <strong>
  • <em>
  • <img>*

* Image elements work a little differently than the others, but we'll get to that

display: inline;

Display one after another in the direction that sentences run.

display: inline;

CAN take side spacing and reflow content around it.
CANNOT reflow content around it with top or bottom spacing. Margin will have no effect on inline elements.

display: inline;

Top and bottom spacing comes from line height on its parent element.

display: inline-block;

The best of both worlds. This is also like how images behave by default.

Watch out for extra spacing, especially on default <img>!
https://dev.to/christiankaindl/the-strange-img-gap-in-html
https://css-tricks.com/fighting-the-space-between-inline-block-elements/

display: inline-block;

They can take spacing on all sizes, and height and width.

Semantics, & Accessiblity

You can always change the display property to fit your layout needs. A button can become block element, a div can become inline element, etc. It's more important to use the proper semantic HTML element, and then change how it displays, rather than choose an element for its default display property.

Break

display: none;

  • Removes an element visually and the space it takes up in the layout so that the page renders as if that element does not exist.
  • Removes the item from the accessibility tree, which means it will not be announced to screen readers. This includes all of its descendants.

A robust class to visually hide an element but preserve it for accessibility

.screen-reader-only {
  border: 0;
  clip: rect(0 0 0 0);
  height: auto;
  margin: 0;
  overflow: hidden;
  padding: 0;
  position: absolute;
  width: 1px;
  white-space: nowrap;
}

https://zellwk.com/blog/hide-content-accessibly/

Centering things

Centering things

Not so stressful if you know which display property you are working with.

display: block;

Center horizontally using auto margins

margin-left: auto;

margin-right: auto;

display: inline;
display: inline-block;

Center horizontally using text-align: center; on the parent element.

Center vertically

Inline & Inline-block

It's not worth it. There's vertical-align: middle; but it's unreliable.


Block

Use positioning trick.

Vertically center an element with positioning

.center-me {
  display: block;
  position: relative;
  top: 50%;
  transform: translateY(-50%);
}
